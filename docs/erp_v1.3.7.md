# ERP 系统 (ERP System) 技术文档

**版本**: v1.3.7  
**最后更新**: 2026-04-03

## 1. 项目简介

本 ERP 系统是一个基于 Electron + Vue 3 的桌面应用程序，聚焦「报销管理」与「差旅管理」。
v1.3.7 版本重点完善差旅模块的单据标准化与行程消费明细能力，提升流程规范性与导出可用性。

核心能力：
- **单据类型标准化**：差旅单据统一为“出差申请、酒店发票、酒店水单、交通工具发票”四类。
- **类型化上传约束**：按单据类型限制上传格式，出差申请仅支持 PDF。
- **行程消费条例**：每个行程可维护多条消费条例（类型、金额、备注）。
- **统计可视化**：行程层面可直接查看消费条例条数与金额汇总。
- **导出字段增强**：差旅导出新增消费条例数量与消费金额。

## 2. 技术栈 (Tech Stack)

- **桌面框架**: Electron (主进程/渲染进程架构)
- **前端框架**: Vue 3 (Composition API) + Vite
- **UI 组件库**: Element Plus
- **可视化**: ECharts
- **数据存储**: Better-SQLite3 + Electron-Store
- **自动更新**: electron-updater
- **CI/CD**: GitHub Actions

## 3. v1.3.7 关键更新

### 3.1 差旅单据标准化
- **类型收敛**：差旅单据类型统一到四类：出差申请（仅 PDF）、酒店发票、酒店水单、交通工具发票。
- **上传策略**：单据管理弹窗新增类型选择，按类型动态限制可上传文件扩展名。
- **历史兼容**：对旧类型字段（如 `ticket`、`车票`、`单据`）做兼容映射，避免历史数据异常。

### 3.2 行程消费条例
- **明细建模**：行程新增 `expenses` 数组字段，支持多条消费明细。
- **明细结构**：每条消费包含费用类型、金额、备注，支持新增/删除和统一保存。
- **交互补齐**：新增“消费条例管理”弹窗与行程列表汇总展示，操作路径更直观。

### 3.3 导出与归档
- **明细导出增强**：差旅导出（CSV/PDF）新增“消费条例数量”“消费金额”列。
- **ZIP 命名一致**：差旅凭证 ZIP 导出按新单据类型命名，同时兼容旧字段导出。

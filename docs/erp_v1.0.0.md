# ERP 系统 (ERP System) 技术文档

**版本**: v1.0.0  
**最后更新**: 2026-03-02

## 1. 项目简介

本 ERP 系统是一个基于 Electron + Vue 3 的桌面应用程序，专注于报销管理功能的初代版本。旨在为 Windows 用户提供便捷的报销条目创建、凭证管理及本地化数据存储解决方案。

系统核心功能包括：
1. **报销条目管理**：用户可新增报销记录。
2. **凭证管理**：强制要求关联实物照片、电子发票、支付记录截图三项凭证。
3. **本地存储**：所有上传的文件及数据存储在用户指定的本地目录，确保数据隐私与安全。
4. **列表展示**：提供直观的报销记录列表视图。

## 2. 技术栈 (Tech Stack)

- **桌面框架**: [Electron](https://www.electronjs.org/)
- **前端框架**: [Vue 3](https://vuejs.org/) (Composition API)
- **UI 组件库**: [Element Plus](https://element-plus.org/)
- **本地数据库**: [Better-SQLite3](https://github.com/WiseLibs/better-sqlite3) (SQLite 数据库，用于存储报销记录元数据)
- **文件处理**: Node.js `fs` 模块 (用于文件复制与管理)
- **配置管理**: [electron-store](https://github.com/sindresorhus/electron-store) (用于存储存储路径等用户配置)

## 3. 业务逻辑说明 (Business Logic)

### 3.1 报销条目 (Reimbursement Item)

用户创建报销条目时，需填写以下基本信息，并上传必要的凭证。

#### 3.1.1 数据模型
每个报销条目包含以下字段：
- `id`: 唯一标识符 (UUID)
- `date`: 报销日期 (默认为当前日期)
- `amount`: 报销金额 (精确到小数点后2位)
- `category`: 报销类别 (如：差旅费、办公用品、招待费等)
- `description`: 费用描述/备注
- `proofs`: 凭证文件路径集合 (JSON 对象)
  - `physical_photo`: 实物照片路径
  - `electronic_invoice`: 电子发票路径
  - `payment_screenshot`: 支付记录截图路径
- `created_at`: 创建时间

#### 3.1.2 凭证上传规则
- **三证齐全**: 系统强制检查三个凭证位（实物照片、电子发票、支付记录截图）是否均已上传文件。若有缺失，禁止提交保存。
- **文件格式**: 支持常见的图片格式 (JPG, PNG) 及 PDF (针对电子发票)。

### 3.2 文件存储与路径配置

#### 3.2.1 存储路径配置
- 用户可在“设置”页面指定一个本地目录作为**根存储路径** (例如 `D:\MyERP_Data`).
- 系统首次运行时，需引导用户设置该路径，或使用默认路径 (如 `Documents/ERP_Data`).
- 该配置持久化存储于 `electron-store` 中。

#### 3.2.2 文件组织结构
在根存储路径下，系统自动按以下结构管理文件：
```
[根存储路径]/
├── db/                     # 数据库文件 (如 erp.db)
├── attachments/            # 附件/凭证存储目录
│   ├── [YYYY-MM]/          # 按年月分文件夹 (可选，避免单目录文件过多)
│   │   ├── [UUID]_physical.jpg
│   │   ├── [UUID]_invoice.pdf
│   │   └── [UUID]_payment.png
│   └── ...
└── config.json             # (可选) 其他配置
```
- **文件重命名**: 上传的文件在保存到本地存储时，会自动重命名以避免冲突，通常使用 `[条目ID]_[凭证类型].[扩展名]` 的格式。

### 3.3 列表展示与交互

- **列表视图**:
  - 以表格形式展示报销条目。
  - 列包含：日期、类别、金额、描述、凭证状态（预览图标）。
  - 支持按日期降序排列。
- **详情查看**:
  - 点击条目可查看详情，并预览三个凭证文件。
  - 支持调用系统默认程序打开文件 (如使用系统看图软件查看大图)。

## 4. 代码架构说明 (Code Structure)

### 4.1 目录结构规划

```
/
├── dist/                   # 构建产物
├── src/
│   ├── main/               # Electron 主进程
│   │   ├── index.js        # 入口文件
│   │   ├── ipc.js          # IPC 通信处理 (文件读写、数据库操作)
│   │   └── database.js     # 数据库初始化与操作封装
│   ├── renderer/           # Vue 3 前端渲染进程
│   │   ├── components/     # UI 组件 (ReimbursementForm, ReimbursementList)
│   │   ├── views/          # 页面 (Home, Settings, Create)
│   │   ├── store/          # Pinia 状态管理
│   │   ├── App.vue
│   │   └── main.js
│   └── preload/            # 预加载脚本 (暴露 safe API 给渲染进程)
├── package.json
└── electron-builder.yml    # 打包配置 (Windows 目标)
```

### 4.2 关键流程

#### 4.2.1 新增报销流程
1. **前端**: 用户填写表单，选择三个文件。
2. **前端**: 点击提交，通过 IPC 发送数据及文件路径给主进程。
3. **主进程**:
   - 读取用户配置的存储路径。
   - 将临时文件复制/移动到 `attachments` 目录，并重命名。
   - 将条目数据 (含新文件路径) 写入数据库。
   - 返回成功响应。
4. **前端**: 收到成功响应，刷新列表，提示用户。

#### 4.2.2 存储路径变更
1. **前端**: 用户选择新目录。
2. **主进程**: 更新配置。
3. **(可选高级功能)**: 询问用户是否将旧目录数据迁移到新目录。

### 4.3 构建配置

- **目标平台**: Windows (`nsis`)
- **配置**: `electron-builder`
- **注意事项**: 需处理好文件权限，确保应用有权读写用户指定的存储目录。
